PROJECT OVERVIEW
----------------
Project Name: AutoFair
Domain: Clinical Genomics Machine Learning
Goal: A 3-stage bias mitigation pipeline addressing representation bias, technical batch effects, and predictive safety.
Dataset: TCGA-BRCA (Breast Invasive Carcinoma) from UCSC Xena.
  1. Phenotypes (Clinical/Metadata)
  2. Curated Survival (Ground truth: Overall Survival)
  3. STAR-TPM (RNA-seq matrix with ~60,000 genes)

STAGE 1: DATA INGESTION & OPTIMIZATION (loader.py)
--------------------------------------------------
Goal: Merge three disparate bioinformatics datasets into a unified machine-learning-ready dataframe without data leakage.

1. Barcode Standardization: 
   - TCGA barcodes were truncated to 15 characters (e.g., 'TCGA-A1-A0SB-01A') to identify the specific patient and primary tumor sample.
   - Dropped duplicate vials (keeping the 'first' instance) instead of averaging them. This is critical to prevent data leakage where two samples from the exact same patient end up split between training and testing sets.
2. Data Transposition: 
   - Transposed the STAR-TPM RNA-seq matrix so that patients are rows and genes (~60k) are columns, matching standard ML input expectations.
3. Biological Transformation: 
   - Safely separated the clinical columns from the genomic columns and applied an np.log1p() mathematical transformation to the gene expression values to keep neural network gradients stable during training.
4. Overlapping Column Fix: 
   - Resolved a Pandas merging collision where both clinical and survival datasets contained a `_PATIENT` column by applying `rsuffix='_surv'` during the inner join.
5. High-Speed Parquet Caching: 
   - Implemented a caching mechanism to save the final merged dataframe (containing >60,000 columns) as a compressed `.parquet` file. This bypasses the massive memory and time cost of re-merging the files on every run, dropping load times to fractions of a second.

STAGE 2: FAIRNESS DETECTION & HEURISTIC SCANNING (nlp_scanner.py)
-----------------------------------------------------------------
Goal: Isolate sensitive demographic and clinical attributes for the proxy detection system.

1. The Zero-Shot NLP Trap: 
   - Initially utilized a HuggingFace `bart-large-mnli` model to scan column names. This failed because a general-purpose LLM hallucinated on clinical abbreviations (flagging biological multi-omics clusters like `_pancan_dnamethyl_brca` as sensitive) and completely missed standalone demographic terms like `gender`.
   - Prevented memory overflow by ensuring the 60,000 gene columns (ENSG) were stripped out before feeding the data to the scanner.
2. The Deterministic Scanner Upgrade: 
   - Replaced the heavy NLP model with a lightweight, 100% accurate Deterministic Heuristic Dictionary utilizing regex patterns (e.g., race, ethnicity, gender, age, menopause).
   - The new scanner successfully evaluated the 203 clinical columns in milliseconds and accurately flagged 12 key targets, including `gender`, `days_to_birth`, `age_at_initial_pathologic_diagnosis`, and `menopause_status`.
   - Implemented a fallback mechanism to inject core targets automatically in case the regex scanner fails.
3. Data Type Sanitization: 
   - Intercepted a Scikit-Learn crash (`TypeError`) caused by mixed data types in the UCSC Xena clinical data. Cast all object/boolean columns to strict strings prior to proxy detection.

STAGE 3: PROXY DETECTION & CORRELATION (proxy_detector.py)
----------------------------------------------------------
Goal: Train localized ML models to determine if non-sensitive clinical data acts as a "proxy" that could allow a neural network to reconstruct scrubbed demographic traits.

Methodological Upgrades:
- Class Imbalance Correction: Switched to `class_weight='balanced'` and `balanced_accuracy_score` to prevent the majority class (Female) from masking the minority class (Male) prediction accuracy.
- Leakage Prevention: Shifted the median imputation of `NaN` values to occur *after* the `train_test_split`.
- Missing Target Handling: Added logic to dynamically drop rows where the target column is `NaN` for that specific iteration, preventing Scikit-Learn crashes.

Results & Insights:
1. The Truth About Gender: Prior to the upgrade, the model claimed 99% accuracy in predicting `gender`. After implementing `balanced_accuracy_score`, the true predictive power plummeted to 50.00%. This proves the previous high score was purely an illusion of class imbalance. The clinical data cannot reliably predict a patient's gender in this cohort without bias.
2. Algorithmic Rediscovery: The detector reverse-engineered the standard TNM clinical staging system natively from the correlations. It identified `pathologic_n` and `pathologic_t` as perfect proxies for `pathologic_stage` (R2 ~0.79 to ~0.91).

STAGE 5: DIMENSIONALITY REDUCTION & BASELINE TRAINING (train_baseline.py)
-------------------------------------------------------------------------
Goal: Compress the genomic dataset and establish a predictive baseline model.

1. Demographic Injection: Forced the pipeline to ingest sensitive demographics (`gender`, `race`, `menopause_status`), using safe string-checking imputation and One-Hot Encoding.
2. Biological Filter: Applied `VarianceThreshold(threshold=0.1)` to strip out static "house-keeping" genes, dropping the dataset from 60,000 down to ~15,000 Highly Variable Genes (HVGs).
3. Mathematical Compression: Applied `PCA(n_components=128)` to compress the remaining HVGs into a dense, neural-network-ready 128-dimension vector (`Gene_PC_1` to `Gene_PC_128`).
4. Baseline Model Serialization: Trained a Random Forest classifier to predict Overall Survival using the clinical targets and compressed genes. Saved the frozen state, features, and a test data slice via `joblib`.

STAGE 6: EXPLAINABLE AI & BIAS DISCOVERY (run_shap.py)
------------------------------------------------------
Goal: Audit the baseline model using SHAP to prove the existence of biases before implementing adversarial mitigation.

1. Modern API Integration: Transitioned to `shap.plots` to bypass legacy interaction grid bugs and generate clean global bar charts and directional beeswarm plots.
2. The "Smoking Gun" (Missing Data Bias): SHAP analysis revealed that the single most important driver of survival predictions was `menopause_status_Missing`. This proved the baseline algorithm was cheating—relying on hospital clerical/administrative artifacts rather than pure biological signals.
3. The Success of PCA: The model heavily utilized `Gene_PC_3` and `Gene_PC_1`, validating that the mathematical compression successfully preserved powerful biological survival patterns.
4. Minority Dilution: Discovered that minority demographics (e.g., male breast cancer patients) are structurally ignored by standard algorithms due to "Information Gain" maximizing for the 99% majority.

NEXT STEPS: PHASE 4 (ADVERSARIAL IN-PROCESSING)
-----------------------------------------------
The pipeline is structurally complete but mathematically biased. The immediate next step is to build a PyTorch Adversarial Architecture. This requires:
1. A Main Predictor for Overall Survival.
2. An Adversarial Head to guess demographics.
3. Sample Reweighing to correct minority dilution.
4. An adversarial penalty loop to force the network to ignore hospital artifacts and rely solely on the biological `Gene_PCs`.
